## The Consultant's Perspective

Arnold Quesenberry described the two primary use cases he sees for Resource Manager across his consulting engagements.

### Complementing Existing MRP

"We have found these products are a perfect complement to traditional MRP and Shop Control systems. They enable the user to quickly generate various 'what-if' plans for feasible and efficient master scheduling, all without disrupting the current system."

This is the [ERP add-on approach](/erp-scheduling-add-on) validated by an independent consultant. Resource Manager does not replace your MRP — it adds the [what-if analysis](/what-if-analysis-manufacturing) and [finite capacity scheduling](/finite-capacity-scheduling-software) that traditional MRP cannot perform. And it does so "without disrupting the current system" — meaning zero risk to existing operations.

### Starting System for New Users

"In addition to complementing traditional systems, the Resource Manager is the perfect beginning system for companies just getting started with computerizing their operations, both here and abroad. It provides a quick, easy, and flexible method to organize parts, routings, operations, costs, Bills-Of-Materials, and perform basic scheduling, inventory management, even purchasing."

For manufacturers that have not yet implemented any production system, Resource Manager provides a comprehensive starting point that covers scheduling, BOMs, routings, costing, inventory, and purchasing — without the cost and complexity of a full ERP implementation.

### The Excel Advantage

"Our customers love the fact that since the Resource Manager is compatible with Excel, they are immediately comfortable with running the system."

This Excel compatibility is not a minor feature — it is the adoption accelerator. When a consultant introduces new software to a client's team, the biggest risk is adoption failure. Excel familiarity eliminates that risk.

## Two Distinct Value Propositions

The consultant's perspective reveals that Resource Manager serves two distinct markets equally well.

### For Manufacturers with MRP: Add What Is Missing

If you already have MRP and shop control systems, Resource Manager adds the capabilities they lack:

- **What-if scenario analysis** — test schedule changes before implementing them
- **Finite capacity scheduling** — see real capacity constraints, not infinite assumptions
- **Visual scheduling** — Gantt charts and load graphs that MRP systems rarely provide
- **Non-disruptive implementation** — works alongside your existing system

### For Manufacturers without Systems: Start Right

If you are still scheduling manually or with basic spreadsheets, Resource Manager provides a comprehensive starting point:

- **Parts and routings organization** — structure your production data properly
- **Bills-of-Materials** — define your product structures
- **Basic scheduling** — finite capacity scheduling from day one
- **Inventory management** — track materials and finished goods
- **Purchasing support** — generate purchase requirements from schedules
- **Costing** — understand your production costs

Starting with Resource Manager is significantly faster and more affordable than starting with a full ERP system, and it builds the data foundation that a future ERP implementation would require anyway.

## Why Consultant Recommendations Matter

When a consultant recommends software across multiple engagements, the recommendation carries weight that vendor marketing cannot match.

**Consultants see many operations.** Arnold Quesenberry works with multiple manufacturers across different industries. His recommendation is based on seeing Resource Manager work in diverse environments, not just one.

**Consultants have no vendor bias.** Unlike ERP vendors who recommend their own scheduling modules, independent consultants recommend what works best for each client. When they consistently recommend the same tool, it reflects genuine effectiveness.

**Consultants value implementation speed.** Consulting engagements are time-limited. Tools that implement quickly and show results fast are preferred over tools that require months of setup. Resource Manager's rapid implementation aligns with consulting timelines.

## Lessons for Your Scheduling Decision

**You do not have to choose between MRP and advanced scheduling.** They complement each other. Keep your MRP for materials planning and add [Resource Manager](/resource-manager-db-2) for capacity scheduling and what-if analysis.

**If you have no system, start with scheduling.** Resource Manager provides a broader starting point than most manufacturers realize — covering scheduling, BOMs, costing, inventory, and purchasing in a single tool.

**Excel compatibility is a strategic advantage.** The fastest path to adoption is leveraging tools your team already knows. Excel compatibility eliminates training time and adoption risk.
